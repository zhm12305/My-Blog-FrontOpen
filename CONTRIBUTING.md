# Contributing

## 分支建议

1. `main`: 生产可发布分支
2. `feature/*`: 新功能
3. `fix/*`: 缺陷修复

## 提交流程

1. Fork / 新分支开发
2. 保持单次提交聚焦
3. 本地执行：
   - `npm install`
   - `npm run build`
4. 提交 PR（使用仓库 PR 模板）

## 代码规范

1. 优先保持与现有 Vue2 风格一致
2. 不要在前端提交任何密钥
3. API 地址变更需同步更新文档

## 变更说明

PR 中建议包含：

1. 变更背景
2. 影响范围
3. 测试结果
4. 回滚方式（如影响生产）
